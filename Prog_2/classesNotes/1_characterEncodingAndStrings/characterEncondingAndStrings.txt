--CHARACTER ENCODING--
As the previous class showed, memory can be understood as a collection of bytes.
A byte, however, represents a number between 0 and 255. Therefore, how characters are represented in a computer?
Through mappings -- tables that relate numbers and characters.
Before starting, lets establish some concepts:
	Character: a language symbol (letter, digit or signal);
	Character set (charset): a set of every characters supported by a system or coding pattern;
	Codification: the process of translating characters in their respective numeric value in a system or coding pattern;

--ASCII ENCONDING--
For digital electronic computers, the oldest character encoding is called American Standar Code for Information Interchange (ASCII).
ASCII contemplates a set of english characters, signals and some control characters, totalizing 128 characters (0~127).
ASCII is supported to every current computer system.
General organization in characters:
	0~31 + 127: control characters;
	32~126: unpredictable characters;as

--ENCODING PAGES--
There is an interesting point in ASCII, it defines 128 characters.
When ASCII encoding was created, it was implemented in 7 bits systems, and with the 8 bits system, the eighth bit was a parity bit. 
But with only 128 symbols, for some contexts, there were symbols missing. Therefore, the encoding pages were created, extending the ASCII table in more 128 symbols according to a certain need.
Examples:
	CP-437: added accented characters and simple graphics.
	KOI8-R:Russian Cyrilic.
	BraSCII: brazilian portuguese

--MULTIBYTE CHARACTERS--
Even with the encoding pages, for the variety of languages and applications, 256 symbols weren't enough.
The idea of multibyte characters, came about as an solution for it, using more than one byte for character.
Tons of encoding patterns were proposed using multibyte characters (2-4 bytes, for example).
One of them was the Unicode pattern. It has an enormous quantity of characters, the most famous ones are:
	UTF-8 (8-bit Unicode Transformation Format): 1 to 4 bytes per character;
	UTF-16 (16-bit Unicode Transformation Format): 2 to 4 bytes per character, utilized in some windows API's;
	UTF-32 (32-bit Unicode Transformation Format): 4 bytes per character;
The most used one is UTF-8. But, why?
	->Because of its adjustable size coding

--CONCEITO DE STRING--
Sequências de caracteres utilizadas para definir informações textuais.
Em C: vetor de caracteres (vetor de char)
O final de uma string é identificada por um caractere nulo ('\0'). Logo, se uma string tem 50 caracteres o vetor deve ter pelo menos 51 posições.

--STRINGS CONSTANTES--
Strigs constantes são sequências de caracteres entre aspas ("").
Há maneiras de definir strings constantes, como referencia-las por ponteiros.

--ESCRITA DE STRINGS--
Para printar uma string, é necessário usar o indicador "%s", e depois passar o ponteiro para a string.
É possível escrever através da função puts também.

--MANIPULAÇÃO DE STRINGS--
Existem bibliotecas para manipular strings (string.h).
	strlen - recebe o ponteiro para uma string como argumento, retorna a quantidade de caracteres dela (sem contar \0)
	strcpy - recebe, em ordem, o ponteiro para onde a string deve ser copiada (vetor de char) e o ponteiro para a string base; retorna o ponteiro da string copiada
	strncpy - funciona igual strcpy, porém recebe um terceiro argumento que indica quantos caracteres devem ser copiados no máximo
	strcat - recebe dois ponteiros para strings, sendo que a segunda deve ser concatenada na primeira. retorna o ponteiro para a string copiada
	strncat- funciona igual strcat. Porém, recebe o terceiro argumento inteiro que indica quantos caracteres devem ser concatenados
	strchr - recebe um ponteiro para string e um caractere qualquer; retorna o ponteiro para a primeira ocorrência do caractere na string fornecida, se não houver, retorna null
	strrchr - recebe um ponteiro para string e um caractere qualquer; retorna o ponteiro p/ última ocorrência do caractere na string fornecida ou NULL se não houver
	strstr - recebe um ponteiro para uma string base e um para uma substring; retorna o ponteiro para a primeira ocorrência da substring na string fornecida ou NULL se não houver
	strdup - recebe um ponteiro p/ uma string e duplica a mesma, alocando a memória necessária.
	strtok - recebe um ponteiro p/ uma string base e um p/ uma string separadora; retorna o ponteiro p a primeira ocorrencia de um caractere na string base antes/após a substring fornecida ou NULL se não houver.
